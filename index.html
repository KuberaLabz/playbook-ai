Setup Guide

01
Install Node & npm

Make sure you have Node.js 18+ installed. Check with node --version.

Terminal
$node --version
$npm --version
02
Install Claude Code

Install globally via npm. You'll need an Anthropic API key from console.anthropic.com.

Terminal
$npm install -g @anthropic-ai/claude-code
03
Set Your API Key

Export your key to your shell environment — add to .bashrc or .zshrc for persistence.

Terminal
$export ANTHROPIC_API_KEY=sk-ant-your-key-here
$echo 'export ANTHROPIC_API_KEY=sk-ant-...' >> ~/.zshrc
04
Navigate to Your Project

cd into any existing project or create a new directory. Claude Code reads your full file tree.

Terminal
$mkdir my-saas && cd my-saas
$claude
05
Give It Context

Type your first instruction. Be specific. The more context you give, the better the output.

Terminal
# Inside Claude Code:
$> Build a REST API with Express.js and PostgreSQL for a task management app with user auth
06
Master the CLAUDE.md File Advanced

Create a CLAUDE.md in your project root. Include: tech stack, coding conventions, business domain glossary, architecture decisions, and things Claude should never do (e.g. 'never use any type', 'always add error handling'). This file is read every session — it's the difference between consistent quality and inconsistent output.

07
Use Parallel Workstreams Advanced

For large projects, open multiple Claude Code sessions simultaneously in different terminals — one working on frontend, one on backend, one on tests. They work in parallel. Merge via git. This multiplies your throughput further.

08
Build a Cost Tracker Advanced

Claude Code uses Anthropic API credits. For large projects, track your spend with: claude --print-cost. Set a daily budget. For client work, factor API costs into your project price (usually $5–$30/project at typical usage).

Setup Guide

01
Install CrewAI

Install CrewAI and its tools package. Python 3.10+ required.

Terminal
$pip install crewai crewai-tools
# Verify install
$python -c 'import crewai; print(crewai.__version__)'
02
Define Your Agents

Create specialist agents with distinct roles, goals, and backstories. The richer the role definition, the better the agent performs.

Terminal
$from crewai import Agent
$from langchain_openai import ChatOpenAI
$llm = ChatOpenAI(model='gpt-4o', temperature=0.1)
$researcher = Agent(
$ role='Senior Research Analyst',
$ goal='Find accurate, current information on any topic',
$ backstory='''You are an expert researcher with 15 years
$ of experience in competitive intelligence. You never
$ rely on a single source. You verify every claim.''',
$ llm=llm,
$ verbose=True
$)
$writer = Agent(
$ role='Expert Business Writer',
$ goal='Transform research into compelling business content',
$ backstory='''Former McKinsey consultant turned writer.
$ You write for senior executives. Dense, precise, no fluff.''',
$ llm=llm
$)
03
Define Tasks

Create specific tasks for each agent. Each task has a description, expected output, and assigned agent.

Terminal
$from crewai import Task
$research_task = Task(
$ description='''
$ Research {topic}. Find:
$ 1. Current market size and growth rate
$ 2. Top 5 competitors with differentiators
$ 3. Key trends in the past 6 months
$ 4. Customer pain points from forums/reviews
$ ''',
$ expected_output='Structured research report with sources',
$ agent=researcher
$)
$writing_task = Task(
$ description='Write a 1,000-word executive brief based on the research.',
$ expected_output='Polished executive brief in Markdown',
$ agent=writer,
$ context=[research_task] # uses research output
$)
04
Assemble the Crew & Run

Create the Crew with your agents and tasks. Kick off the pipeline.

Terminal
$from crewai import Crew, Process
$crew = Crew(
$ agents=[researcher, writer],
$ tasks=[research_task, writing_task],
$ process=Process.sequential, # or hierarchical
$ verbose=True
$)
$result = crew.kickoff(inputs={'topic': 'AI chip market 2025'})
$print(result.raw)
05
Add Tools

Give agents tools: web search (Tavily), file read/write, code execution, database queries. Tools dramatically expand what a crew can do.

Terminal
$from crewai_tools import TavilySearchResults, FileWriterTool
$researcher = Agent(
$ ...,
$ tools=[TavilySearchResults(), FileWriterTool()]
$)
06
The Backstory Is the System Prompt Advanced

In CrewAI, the agent's backstory is functionally its system prompt. A generic backstory like 'You are a researcher' produces generic research. A specific backstory like 'You are a former Goldman Sachs equity analyst with 12 years covering the technology sector. You never cite a claim without a source. You prioritise primary data over secondary analysis. You flag uncertainty explicitly.' produces dramatically better output. Spend 30 minutes per agent on the backstory — it's the highest-leverage tuning you can do.

07
Use Process.hierarchical for Complex Tasks Advanced

For tasks with dynamic, unpredictable workflows, switch from Process.sequential to Process.hierarchical. This adds a manager LLM that decides which agent to call based on the current state of the task — not a fixed sequence. Use sequential for predictable pipelines (research → write → edit). Use hierarchical for complex tasks where the path depends on what's discovered along the way.

08
Deploy as a REST API Advanced

Wrap your CrewAI crew in a FastAPI endpoint so it's callable from anywhere: your n8n automation, your Make scenario, your client's dashboard. The endpoint receives inputs, kicks off the crew, and returns the output. A deployed crew API is a product — you can charge per API call, per month, or as part of a managed service. Without the API wrapper, it's just a script.

Operator Insights

Role Quality = Output Quality
The biggest mistake in CrewAI is vague agent roles. 'Research Agent' is weak. 'Senior Competitive Intelligence Analyst with 15 years at Gartner who never relies on a single source' is powerful. The backstory trains the behaviour.
Crews Don't Sleep
A well-built CrewAI content system runs 24/7. Deploy it on a cron job. While you sleep, your crew is researching, writing, and queuing content for the next day.
Hierarchical Process Is Underused
Most tutorials use sequential process. Hierarchical process (with a manager LLM) handles complex, dynamic tasks far better. Learn it — it's what separates amateur CrewAI from production CrewAI.
